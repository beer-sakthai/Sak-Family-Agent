# Spike

A spike is a focused, time-boxed experiment to validate an approach before committing to a full build.

## When to Spike

- You're unsure if an approach will work
- You need to test an API or framework
- You want to compare two approaches

## Rules

- Time-box: max 1 hour
- Code is throwaway (may not be merged)
- Document findings for later build
- If the spike proves the approach, move to full build
