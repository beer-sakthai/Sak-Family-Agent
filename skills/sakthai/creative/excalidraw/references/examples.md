# Excalidraw Examples

This file is a quick reference for the Editor to recall the various Excalidraw diagram patterns and structures.

## Architecture Diagram
- First, show the system boundarx (the box that encapsulates the whole system)
- Next, list major components (services, databases, caches, queues) as nodes
- Then, show connections with arrows indicating data flow
- Finally, add a legend and labels for clarity

![Architecture Example](https://excalidraw.com/#/api/download?filename=architecture.excalidraw)

## Sequence Diagram
- Use actors/lifelines to represent entities
- Use solid arrows for synchronous calls
- Use dashed arrows for asynchronous calls
- Add titles and conditional notes along the arrows

## Flow Diagram
- Use diamonds for start/end points
- Use rectangles for process steps
- Use diamonds for decisions/conditions
- Use arrows for flow
